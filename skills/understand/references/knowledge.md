# 위키형 지식 베이스 추출 가이드라인 (understand-knowledge)

* **Parent (상위 스킬)**: [[skills/understand/SKILL.md]]

---


본 지침은 카파시(Karpathy) 패턴 LLM 위키(3계층 지식 베이스 구조: Raw Sources, Wiki Markdown, Schema 파일)를 정밀 분석하고 추출하여, 상호 연결 관계 및 지식 군집화(Topic Clustering)가 적용된 지식 그래프 대시보드를 구축하는 제어 가이드라인입니다.

## 카파시 LLM 위키 패턴 요소 감지 (What It Detects)

카파시 LLM 위키 패턴은 다음과 같은 고유 정적 형식을 가집니다.
- **원천 자료 (Raw Sources)**: 불변의 원본 가이드 문서, 논문 및 데이터 파일
- **위키 (Wiki)**: LLM이 자율 생성한 마크다운 문서들로, 내부에서 위키링크(`[[대상문서]]` 문법)를 활용하여 상호 참조
- **스키마 (Schema)**: `CLAUDE.md`, `AGENTS.md` 등과 같이 개발 및 운영 규칙이 정의된 전역 구성 파일
- **인덱스 (index.md)**: 전체 지식들을 카테고리별로 정연하게 분류한 카탈로그 파일
- **로그 (log.md)**: 시간 흐름에 따른 운영 이력을 기록한 변경 로그

해당 패턴 감지 시그널: `index.md` 파일이 존재하고, 본문 내에 대량의 위키링크(`[[...]]`)를 포함한 여러 마크다운 파일들이 존재하며, `raw/` 하위 폴더와 전역 에이전트 스키마 설정 파일을 동반하고 있는 구조.

## 세부 단계별 분석 지침 (Instructions)

### 단계 1: 감지 및 파싱 실행 (DETECT)

1.  **분석 타겟 디렉토리 파악**:
    - 사용자가 인자값으로 경로를 입력했다면 해당 경로를 기준점으로 삼으십시오.
    - 입력 경로가 없다면 현재 작업 디렉토리(CWD)를 분석 기점으로 간주합니다.
2.  **구조 진단 및 파싱 스크립트 가동**:
    - 이관에 동봉된 파이썬 스크립트를 파악하여 기동하십시오.
      ```bash
      python3 <SKILL_DIR>/parse-knowledge-base.py <TARGET_DIR>
      ```
    - 만약 해당 스크립트 실행이 에러로 종료된다면, 타겟 디렉토리가 카파시 표준 위키 규격을 충족하지 못함을 친절한 한국어로 공지하고 요구 스펙을 요약해 주십시오.
    - 정상 종료 시, 기동된 스크립트는 분석 중간 요약본인 `scan-manifest.json` 파일을 `<TARGET_DIR>/.understand-anything/intermediate/` 경로에 기록합니다.
3.  **진단 매니페스트 확인 및 개요 공지**:
    - 생성된 `scan-manifest.json` 파일을 열어 다음 지표를 확보하고 사용자에게 친절하게 요약해 주십시오.
      - "N개의 아티클, N개의 원본 파일, N개의 토픽 및 N개의 위키링크(이 중 N개 미해결)를 지닌 카파시 위키 구조를 완벽히 인지했습니다."
      - `index.md`에서 파생된 메인 카테고리 분류 일람 공지

### 단계 2: 정적 구조화 완료 (SCAN)

- 단계 1에서 기동한 파이썬 스크립트가 이미 결정론적인(Deterministic) 스캔을 기지 완료해 둔 상태입니다. `scan-manifest.json` 내부에는 다음과 같은 완성형 노드 및 관계 정보가 포함되어 있습니다.
  - 아티클 노드 (개별 마크다운 문서 대응, 위키링크 및 헤더 구성, Frontmatter 추출 포함)
  - 원천 자료 노드 (`raw/` 하위 물리 파일 대응)
  - 토픽 노드 (`index.md` 구조의 섹션 대분류 및 소분류 대응)
  - `related` 관계 엣지 (위키링크 상호 참조 대응)
  - `categorized_under` 관계 엣지 (카테고리 분류 구조 대응)
- 추가적인 디렉토리 스캔을 낭비할 필요 없이, 해당 매니페스트 데이터를 기반으로 **단계 3**으로 진행하십시오.

### 단계 3: 심층 지식 추론 및 정밀 분석 (ANALYZE)

본 정적 정보 외에, 텍스트 배후의 암묵적인 교차 지식을 추론하기 위해 `article-analyzer` 서브에이전트 군집을 동시 기동하여 분석 효율을 극대화합니다.
1.  `scan-manifest.json`에서 분석 대상 아티클 목록 전체를 확보하십시오.
2.  약 10~15개의 아티클을 하나의 분석 배치(Batch) 단위로 묶으십시오. 이때 동일 카테고리에 속한 아티클들을 하나로 묶으면 상호 연관 암묵 관계를 LLM이 추론하는 효율이 비약적으로 향상됩니다.
3.  개별 배치를 전담 수행할 `article-analyzer` 서브에이전트들을 기동하십시오. 전달 데이터:
    - 배치 대상 아티클 정보 (ID, 타이틀, 요약본, 추출 위키링크, 소속 카테고리 및 실물 마크다운 내용 등)
    - 이미 생성되어 있는 전체 노드 ID 리스트 (상호 참조 검증용)
    - 임시 배치 저장용 인덱스 번호 및 출력용 디렉토리 경로: `intermediate` 하위 폴더
    - 에이전트는 각각 실행 결과를 `analysis-batch-{N}.json` 파일로 중간 저장소에 기록합니다.
4.  동시성에 따라 최대 3개의 분석 배치를 병렬 실행하고, 모든 서브에이전트 태스크가 백그라운드에서 완료될 때까지 안전하게 대기하십시오.
5.  만약 특정 배치의 분석 작업이 부분 실패하더라도, 경고 로그를 기록하되 흐름을 전면 차단하지 말고 확보된 정적 매니페스트를 최종 가공하여 복구 탄력성(Fault-tolerance)을 유지하십시오.

### 단계 4: 그래프 데이터 병합 (MERGE)

1.  동봉된 병합 전용 스크립트를 안전하게 호출하여 기동하십시오.
    ```bash
    python3 <SKILL_DIR>/merge-knowledge-graph.py <TARGET_DIR>
    ```
2.  기동된 병합 스크립트는 내부적으로 다음 정합성 보정을 완수합니다.
    - 정적 매니페스트 데이터와 각 서브에이전트들이 생성해 낸 `analysis-batch-*.json` 데이터 병합
    - 동일 개체에 대한 중복 노드 제거 (대소문자 구분 배제 명칭 대조 등)
    - 노드 및 엣지 타입 표준 규격 단일화 (노멀라이징)
    - `index.md` 구조에 따른 아키텍처 아웃라인 레이어 빌드 및 순차 가이드 투어 생성
    - 병합 완료 후 `assembled-graph.json`을 중간 저장소에 출력합니다.
3.  병합 리포트를 수집하여 최종 병합 결과를 요약해 주십시오. (총 노드/엣지수, LLM이 추가 추론해낸 지식 수 등)

### 단계 5: 유효성 검증 및 자산 배포 (SAVE)

1.  임시 저장된 `assembled-graph.json` 데이터의 정합성을 최종 탐색하십시오.
2.  정적 가드레일 정합성 검사 기동:
    - 모든 관계(Edge)의 `source` 및 `target`이 실제로 그래프 상에 살아 존재하는 노드 ID를 가리키고 있는지 전수 검사
    - 모든 노드가 필수 5대 속성(`id`, `type`, `name`, `summary`, `tags`, `complexity`)을 온전히 채우고 있는지 검증
    - 연결 고리가 끊어진 관계 데이터(Dangling edges) 발견 시 예외 없이 완전 차단 소거
3.  검증 통과된 그래프를 `<TARGET_DIR>/.understand-anything/knowledge-graph.json` 위치에 배포 복사하십시오.
4.  분석 실행 시간 및 Git 정밀 지표를 결합한 가벼운 `meta.json` 메타데이터를 저장하십시오.
5.  중간 가분석 임시 파일이 담긴 `intermediate/` 하위 폴더는 잔여 용량을 절약하기 위해 원천 삭제 처리하십시오.
6.  확보된 통계 정보(N개의 아티클, N개의 엔티티, N개의 레이어 구조 및 연관성 총량)를 한글로 가독성 있게 요약 선언하십시오.
7.  시각적으로 즉시 탐색할 수 있도록 대시보드 스킬을 자동 구동 제안하십시오.
    ```
    /understand-dashboard <TARGET_DIR>
    ```

## 운영 고려사항 (Notes)

- 파이썬 기반 정적 파서가 이미 위키링크 및 카테고리 관계의 대부분을 기계적으로 정확히 추출해 냅니다. LLM 분석 에이전트들은 오직 정적 코드 상으로 분석 불가능한 '암묵적인 시맨틱 관계 지식'을 추론하는 특화 용도로만 콤팩트하게 쓰입니다.
- 카테고리 정의 및 설계 위계는 물리 파일의 디렉토리 깊이나 특수 파일명 접두사가 아닌 오직 `index.md`에 명기된 논리 하이러키 구조만을 SSOT(Single Source of Truth) 기준으로 신뢰하여 해석합니다.
- 본 지식 그래프는 시각화 대시보드 화면상에서 수평/수직 다이어그램이 아닌 자유로운 포스 레이아웃(Force-directed layout)으로 부드럽게 펼쳐지도록 `kind: "knowledge"` 특화 메타 옵션을 결합하여 빌드됩니다.
- `raw/` 하위의 바이너리 파일이나 PDF 등은 파싱하느라 시간을 지체하지 않고 파일명과 대략적 크기 정보만 노드로 단순화하여 가볍게 관계 매핑합니다.
