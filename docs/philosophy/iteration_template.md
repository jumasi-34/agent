# [Philosophy] Iteration Template

## 목적

Iteration은 프로젝트의 최소 개발 단위이다.

모든 작업은 하나의 명확한 목표를 가진 Iteration으로 수행한다.

한 Iteration에서는 하나의 문제만 해결한다.

Iteration 범위를 벗어나는 구현은 수행하지 않는다.

---

# Iteration 정보

Iteration 번호

Iteration 이름

Phase

작성일

작성자

상태

* Planned

* Review

---

# 목적

이번 Iteration에서 해결하려는 문제를 정의한다.

왜 이 작업이 필요한지 설명한다.

---

# 완료 기준 (Definition of Done)

이번 Iteration이 완료되기 위한 조건을 정의한다.

예)

* 문서 작성 완료
* 설계 완료
* 테스트 완료
* 리뷰 완료

---

# Scope

이번 Iteration에서 수행하는 작업만 작성한다.

범위를 명확하게 제한한다.

---

# Out of Scope

이번 Iteration에서 수행하지 않는 작업을 작성한다.

예)

* Router 구현

* Agent 구현

* Rule 수정

* Component 개발

---

# 사전 확인 사항

작업 전 반드시 확인한다.

* Vision
* Principles
* Charter
* Roadmap
* 관련 Rule
* 관련 Agent
* 관련 Skill

---

# 현재 구조 분석

현재 프로젝트 구조를 분석한다.

다음 내용을 기록한다.

* 현재 구조

* 문제점

* 영향 범위

* 기존 자산

---

# 작업 계획

이번 Iteration에서 수행할 작업을 순서대로 작성한다.

예)

1. 분석

2. 설계

3. 문서 작성

4. 검토

---

# 산출물

이번 Iteration에서 생성되는 결과물을 작성한다.

예)

* 문서

* 설계

* 코드

* Rule

---

# 영향 분석

이번 작업으로 영향을 받는 요소를 작성한다.

Agent

Rule

Skill

Prompt

Context

Component

---

# 위험 요소

예상되는 문제를 작성한다.

예)

* 기존 Agent와 충돌

* Rule 중복

* 구조 변경

---

# 검토 결과

Reviewer가 확인할 내용을 작성한다.

예)

* Rule 위반 여부

* Scope 준수 여부

* 기존 구조 영향

---

# 회고 (Retrospective)

Iteration 종료 후 작성한다.

이번 작업에서

잘된 점

개선할 점

배운 점

---

# 다음 Iteration 추천

다음으로 수행할 작업 하나만 추천한다.

반드시 Roadmap을 기준으로 작성한다.
