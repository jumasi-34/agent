# CQ-BI Design Philosophy

## 1. 문서 목적
CQ-BI는 제조 및 품질 데이터를 기반으로 데이터를 탐색, 비교, 판단을 지원하는 복합 분석 시스템입니다.
> 보기 좋은 화면보다 빠르게 이해하고 정확하게 판단할 수 있는 화면을 만든다.

## 2. CQ-BI의 제품 성격
- Manufacturing BI
- Quality Analysis Platform
- Monitoring System
- Data Exploration Tool
- Enterprise Admin System
- Documentation and Knowledge Interface

## 3. 핵심 디자인 원칙 (Core Design Principles)
- **Function over Decoration:** 장식보다 기능을 우선한다.
- **Information First:** 사용자가 생각하고 분석하는 순서로 구성한다.
- **Consistency over Creativity:** 같은 문제는 항상 같은 컴포넌트와 패턴으로 해결한다.
- **Density with Clarity:** 분석 시스템으로서 정보 밀도를 유지하되 명확성을 확보한다.
- **Progressive Disclosure:** 사용자의 필요에 따라 정보를 점진적으로 제공한다.
- **Accessibility:** 색상 외의 인코딩을 병행하여 접근성을 확보한다.
- **Performance Matters:** 느린 화면은 디자인 품질이 낮은 화면으로 간주한다.

## 4. Reference Philosophy
CQ-BI는 특정 제품을 복제하지 않고, 각 레퍼런스의 디자인 철학과 문제 해결 방식을 참고하여 도메인에 맞게 재구성합니다.
- 핵심 우선순위: IBM Carbon(레이아웃/컴포넌트), Linear(탐색/인터랙션), Metabase(분석), IBM Data Visualization(차트), Notion(타이포그래피) 순.
