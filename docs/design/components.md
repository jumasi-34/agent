# CQ-BI Components Reference

CQ-BI는 같은 문제를 항상 같은 컴포넌트와 패턴으로 해결합니다.

## 1. 컴포넌트 목록
| Component | 주요 Reference |
|---|---|
| Sidebar | Linear |
| Global Navigation | Linear |
| Page Layout | IBM Carbon |
| Grid System | IBM Carbon |
| Filter Bar | IBM Carbon + Linear |
| Search | Linear |
| KPI Card | Metabase + Stripe |
| Chart Card | Metabase |
| Data Visualization | IBM Data Visualization |
| Table | IBM Carbon + GitHub + Airtable |
| Badge | GitHub |
| Form | IBM Carbon |
| Monitoring Panel | Grafana |
| Empty State | Vercel |
| Loading State | Vercel |
| Documentation | Notion |
| Typography | Notion + IBM Carbon |
| Detail Drawer | Stripe |
| Alert / Status | Grafana + IBM Carbon |

## 2. 디자인 적용 원칙
- **Consistency:** 새 페이지마다 새로운 UI 패턴을 만들지 않는다.
- **Density:** 정보 밀도를 높일 때는 계층 구조를 명확히 하고, 중요 정보와 보조 정보를 구분한다.
- **Progressive Disclosure:** 상세 정보는 추가 행동(Drill-down 등)을 통해 제공한다.
