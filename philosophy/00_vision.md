# AI Agent Platform Vision

## North Star

> **모든 개발은 품질 부서와 공장 운영 인원의 업무 효율을 높이고, 신뢰할 수 있는 데이터 기반 의사결정을 지원하는 방향으로 이루어진다.**

이 문장은 프로젝트의 최우선 기준이다.

새로운 기능, Agent, Rule, Router, Component, Dashboard를 설계하거나 구현할 때는 항상 이 기준에 부합하는지 먼저 판단한다.

기술적인 완성도보다 **현업에서 실제로 도움이 되는 플랫폼**을 만드는 것을 우선한다.

---

# Vision Statement

우리가 구축하려는 것은 범용 AI 개발 플랫폼이 아니다.

**품질 부서와 공장 운영 인원이 신뢰할 수 있는 데이터 기반 의사결정을 지원하는 AI 기반 품질 분석 플랫폼(Quality Analytics Platform)** 을 구축하는 것이 목표이다.

AI는 단순히 코드를 생성하는 도구가 아니라, 플랫폼을 지속적으로 설계·개발·개선·운영하는 개발 파트너로 동작한다.

이 플랫폼은 품질 담당자와 공장 운영 인원이 필요한 정보를 쉽고 빠르게 확인하고, 반복적인 분석 업무를 줄이며, 동일한 기준으로 데이터를 해석할 수 있도록 지원해야 한다.

---

# Mission

프로젝트의 미션은 품질 데이터를 효과적으로 활용할 수 있는 분석 플랫폼을 구축하는 것이다.

플랫폼은 다음을 지원한다.

* 품질 데이터를 한곳에서 확인할 수 있는 Dashboard
* 공정 및 생산 현황 분석
* 품질 KPI 모니터링
* 불량 원인 분석
* LOT 및 검사 결과 분석
* 데이터 기반 의사결정 지원
* 반복 업무 자동화
* 일관된 사용자 경험 제공

---

# Target Users

이 플랫폼은 다음 사용자를 위해 개발된다.

### Primary Users

* 품질 담당자
* 품질 관리자
* 품질팀 리더

### Secondary Users

* 생산 관리자
* 공장 운영 인원
* 제조 현장 관리자

모든 기능은 이 사용자들이 별도의 설명 없이도 쉽게 이해하고 사용할 수 있는 것을 목표로 한다.

---

# Scope

플랫폼은 다음 영역을 중심으로 발전한다.

## 1. Quality Analytics Platform

품질 데이터를 분석하고 시각화하는 Dashboard와 분석 페이지를 구축한다.

예시

* 품질 KPI Dashboard
* 불량 분석
* 공정별 품질 현황
* LOT 분석
* 검사 결과 분석
* 이상 발생 추이
* 품질 지표 비교

---

## 2. Reusable Development Platform

개발 과정에서 생성되는 모든 자산은 재사용 가능한 형태로 관리한다.

관리 대상

* Dashboard
* Analysis Page
* UI Component
* Chart
* Filter
* Table
* SQL
* Data Model
* KPI 정의
* Prompt
* Rule
* Skill

---

## 3. Design System

일관된 사용자 경험을 위한 디자인 시스템을 구축한다.

* Typography
* Icon
* Chart Palette

* Spacing

---

## 4. Automation

반복적인 품질 업무를 자동화한다.

* KPI 계산
* 이메일 리포트
* 스케줄링

# Core Principles

## User First

현업에서 실제 도움이 되는 기능을 가장 우선한다.

새로운 기술보다 업무 효율 향상을 중요하게 생각한다.

---

## Platform First

개별 기능보다 장기적으로 확장 가능한 플랫폼을 구축한다.

---

## Reuse Before Create

새로운 기능을 만들기 전에 기존 자산을 먼저 활용한다.

재사용은 프로젝트의 기본 원칙이다.

---

## Rule Driven Development

모든 구현은 프로젝트 Rule을 기반으로 수행한다.

Agent는 Rule을 따르며, Rule보다 우선하지 않는다.

---

## Incremental Evolution

프로젝트는 작은 Iteration을 통해 지속적으로 발전한다.

한 번에 전체를 변경하지 않는다.

---

## Knowledge First

모든 개발 경험은 프로젝트의 자산이다.

지식은 문서화되고, Rule과 Agent 개선으로 이어져야 한다.

---

# AI의 역할

AI는 품질 부서와 공장 운영 인원을 위한 개발 조직의 일원으로 행동한다.

AI는 다음 역할을 수행한다.

* 분석 플랫폼 설계
* 데이터 모델 설계
* Dashboard 개발
* 재사용 가능한 Component 개발
* 디자인 시스템 유지
* 데이터 분석 및 집계 지원
* 품질 개선을 위한 구조 제안
* 자동화 기능 개발
* 코드 및 문서 품질 유지

AI는 현업의 의사결정을 대신하지 않는다.

AI의 역할은 **품질 부서와 공장 운영 인원이 더 빠르고 정확하게 의사결정할 수 있도록 지원하는 플랫폼을 구축하는 것**이다.

---

# Success Criteria

프로젝트는 다음 상태를 목표로 한다.

* 품질 부서와 공장 운영 인원이 필요한 정보를 빠르게 확인할 수 있다.
* Dashboard와 분석 페이지를 일관된 방식으로 지속적으로 확장할 수 있다.
* 대부분의 기능이 재사용 가능한 자산으로 관리된다.
* AI Agent가 역할을 분담하여 안정적으로 플랫폼을 발전시킨다.
* 반복적인 개발과 운영 업무가 지속적으로 자동화된다.
* 프로젝트는 시간이 지날수록 더 많은 지식과 경험을 축적하여 발전한다.

---

# Final Vision

우리가 만드는 것은 단순한 Dashboard가 아니다.

우리가 만드는 것은 **품질 부서와 공장 운영 인원이 신뢰할 수 있는 데이터 기반 의사결정을 수행할 수 있도록 지원하는 AI 기반 품질 분석 플랫폼**이다.

모든 Agent, Rule, Skill, Router, Component, Dashboard는 이 목표를 달성하기 위한 수단이며, 모든 의사결정은 반드시 이 비전에 부합해야 한다.
