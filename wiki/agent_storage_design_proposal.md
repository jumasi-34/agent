# [Wiki] 에이전트 데이터 저장소(.agent-storage) 설계 및 마이그레이션 결과

본 문서는 프로젝트 루트 디렉터리에 산발적으로 생성되던 에이전트 및 플러그인 전용 데이터 폴더를 별도의 독립적인 루트 폴더로 격리하고, Git 형상 관리(Push & Pull)에서 영구 배제하도록 구조화한 마이그레이션 결과 보고서입니다.

---

## 1. 개정 설계 배경 및 원칙

기존에는 `.agents/storage/` 하위 보관을 제안하였으나, **"에이전트 제어 폴더(.agents)의 경량화 유지"** 및 **"명시적으로 에이전트가 관리하는 데이터 저장소임을 루트에 각인"**하기 위해 사용자의 피드백을 수렴하여 설계를 개정하였습니다.

1. **에이전트 전용 데이터 디렉터리 신설**: 프로젝트 루트에 `.agent-storage/`를 생성하여 물리 데이터를 완전히 이관합니다.
2. **이모지 및 프로덕션 오염 제로**: 소스 코드 주석 및 마크다운 전체에 이모지를 배제하고, `app/` 소스 코드 수정 없이 순수 환경 격리를 실현합니다.
3. **루트 내 에이전트 링크/잔재물 완전 청소 (Zero-Symlink 및 물리 이관 완료)**: 루트 폴더에 잔존하던 모든 심볼릭 링크 및 지름길 폴더조차 완벽하게 소거하여, 루트 수준에서의 어떠한 이중 노출도 생기지 않도록 완결했습니다.
4. **듀얼 깃 푸시(Dual Git Push) 안전 차단**: `.gitignore` 파일에 에이전트 저장소 영역을 등록하여 푸시 및 풀 과정에서 무거운 바이너리 및 중간 데이터가 커밋되거나 공유되는 문제를 차단합니다.
5. **루트 내 에이전트 잔재물 전수 정비 (2차 정밀 청소 완료)**: 무질서하게 흩어져 있던 에이전트 제작용 스크립트, 예제 파일, 잔여 로그, 스크래치 폴더를 모두 `.agent-storage/` 아래로 정밀 이관 완료하였습니다.

---

## 2. 완료된 폴더 구조 (As-Is vs. To-Be)

마이그레이션 및 루트 레벨 지름길 정리까지 완전히 정비된 현재 워크스페이스의 실제 폴더 트리는 다음과 같습니다.

```text
/home/jumasi/workstation/ (Workspace Root)
├── app/                        # [Production] 실제 구동 소스 코드 영역 (엄격 보존)
├── app.py                      # [Production] 메인 엔트리포인트
├── ... (기타 pyproject.toml, Makefile 등 프로젝트 설정 파일)
│
├── .agents/                    # [Agent OS] 규칙 및 스킬셋 등의 정적 정의 제어판
│   ├── tools/                  # 에이전트 제어 전용 유틸리티 도구함
│   │   ├── link_validator.py
│   │   ├── lint_markdown.py
│   │   └── label_graphify_communities.py  # (이관 완료) 기존 root/tools에서 이동
│   └── wiki/                   # 프로젝트 관련 축적 지식 위키
│       ├── agent_storage_design_proposal.md
│       └── COMMUNITY_CATALOG.md # (이관 완료) 기존 root/COMMUNITY_CATALOG.md에서 이동
│
└── .agent-storage/             # [Agent Storage - 노출] 에이전트 물리 동적 데이터 저장소
    ├── .superpowers/           # (물리 이관 완료) superpowers 세션 상태 데이터
    ├── .understand-anything/   # (물리 이관 완료) understand-anything 분석 데이터
    ├── graphify-out/           # (물리 이관 완료) graphify 지식 그래프 산출물
    │   └── .graphify-community-labels.example.json # (이관 완료) 루트에서 이동
    └── scratch/                # (물리 이관 완료) 에이전트 생성 임시 코드 및 로그 저장소
        ├── test.py             # (이관 완료) 루트에 노출되었던 임시 테스트 파일
        └── ... (기타 23개의 구버전 임시 분석 및 이력 스크래치 파일 일체)
```

> [!NOTE]
> 루트 수준에 가상으로 연결해 두었던 심볼릭 링크 폴더들(`.superpowers`, `.understand-anything`, `graphify-out`, `scratch`)은 사용자 피드백에 따라 **전부 물리적으로 제거**되었으며, 이제 에이전트 관련 모든 물리 폴더와 하위 요소들은 오직 오리지널 경로인 `.agent-storage/` 내부에서만 관리 및 보관됩니다.

## 3. Git 형상 관리 격리 조치 (.gitignore 반영)

듀얼 깃 푸시 시 대용량 파일과 로컬 세션 정보가 레포지토리에 전파되는 것을 차단하기 위해 [.gitignore](.gitignore)의 인공지능 관련 관리 구역을 다음과 같이 정교하게 업데이트했습니다.

```gitignore
# 로그 파일/임시파일
logs/
trash/

# 인공지능 관련 파일 : 별도의 레포로 관리
.agents/

# 에이전트 전용 로컬 동적 데이터 저장소
.agent-storage/
```

이 조치로 인해:
- `.agent-storage/` 폴더 내부의 대용량 그래프 및 모든 상태 데이터는 **Git 트래킹에서 영구적으로 제외**되어 Push & Pull을 타지 않습니다.
- 깃 상태가 항상 완벽하게 청결한 상태로 유지됩니다.

---

## 4. 도입 기대 효과

- **에이전트 제어판(.agents)의 정체성 보존**: 규칙, 지식, 워크플로우 등 순수 설정 코드만 담겨 있어 `.agents` 폴더의 용량이 가볍고 명확하게 유지됩니다.
- **물리적 데이터 일관성**: 에이전트에 의해 자동 생성되는 모든 가공 산출물들이 `.agent-storage/` 폴더 하나에 응축되므로 디렉터리 용량 분석 및 로컬 디바이스 정리가 용이해집니다.
- **최고의 가독성**: 루트에 가상으로 연결되었던 심볼릭 링크 폴더들까지 전수 제거하여, 프로젝트 루트가 이중 노출 없이 극도로 정갈하고 깨끗하게 완성되었습니다.
